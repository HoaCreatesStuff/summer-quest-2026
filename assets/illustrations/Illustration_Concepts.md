# NYC Summer Birthday

## Editorial Icon Concept Library

These concepts define the **hero icon** and its **silhouette hook** for each quest.

The goal is **instant recognition at 48–64 px**.

Every icon should have:

- One hero object
- One memorable silhouette hook
- No unnecessary details
- No repeated hero imagery
- No reliance on micro-details

The silhouette should communicate the concept before color, texture, or rendering.

## Design Rules

- One hero object only.
- One silhouette hook only.
- Personality must be readable at 48 px.
- Do not rely on tiny accessories or hidden details.
- Do not create miniature scenes.
- Remove any detail that does not improve recognition.
- Prioritize silhouette over rendering.
- Prioritize bold shapes over intricate details.
- Every hero icon must have a unique silhouette.
- The collection should feel cohesive when viewed as a complete 5×5 grid.

---

## Work Prompt

Read both the **Illustration Design System** and this **Illustration Concept Library** before generating artwork.

Generate only the first six icons.

Use them to establish the final visual language before continuing with the remaining icons.

Present each icon:

1. Large for design review.
2. At actual 48 px.
3. Placed inside the real scorecard mockup.

Do not continue with the remaining icons until the first six have been approved.

Always optimize for recognition at mobile size rather than visual detail.
