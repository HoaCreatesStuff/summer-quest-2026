# Design System

## Brand Personality

The NYC Summer Quest app should feel:

- Warm
- Curious
- Editorial
- Joyful
- Thoughtful
- Celebratory
- Social
- Premium, not luxurious
- Playful, not childish

---

## Visual Direction

**70% Summer Field Journal**  
**30% Modern Editorial**

The experience should feel like a premium board game that celebrates exploring New York City with friends.

Avoid looking corporate, overly polished, or childish.

---

## Typography

### Headings

- Libre Baskerville
- Italics reserved for emphasis (e.g. *Summer*)

### UI / Body

- Montserrat
- Light: body copy
- Medium: buttons
- SemiBold: labels

## Color Usage

Category colors should remain consistent throughout the app.

- Yellow = Experience NYC
- Green = Community
- Coral = Challenges
- Champagne = Final Quest

Use category colors for:

- Quest cards
- Category badges
- Progress indicators
- Highlights
- Buttons when appropriate

Use neutral colors for:

- Layout
- Containers
- Navigation
- Backgrounds
- Supporting UI

The Final Quest should always be visually distinct through its material treatment rather than size.

---

## Components

The design system applies consistently across:

- Hero
- Progress Card
- Mission Briefing
- Quest Card
- Scoreboard
- Bottom Sheet
- Upload Widget
- Rank Badge
- Progress Bar
- Dialogs
- Toasts
- Empty States
- Success Screens

Every component should inherit the universal color palette.

Category color should communicate quest type before the user reads the title.

---

## Border Radius

- 24–30px on major surfaces
- Soft, friendly corners
- Avoid sharp edges

---

## Shadows

- Soft offset shadows
- Warm shadow color
- Minimal blur
- No heavy floating cards

---

## Motion

- Gentle easing
- Progress animates smoothly
- Bottom sheet slides naturally
- Cards subtly lift on hover
- No bouncy or cartoon-style animations

---

## Layout

- Mobile-first
- 2-column quest grid
- Large touch targets
- Generous whitespace
- Strong visual hierarchy
- Consistent spacing between components

---

## Guiding Principle

> **The interface should disappear as memories appear.**
